Major Categories of Integration

1. Build Time Integration

This is what most of the code implemented today. The container will install the components as libraries, similar to the libraries you install from npm.

Easy to implement

Hard to use different technologies

Size of final package will be big because it contains all the dependencies

You have to deploy again for any changes in dependencies

Tight coupling between container and all Micro-Frontends

Redundancy of common dependency

Major Categories of Integration

2. Run Time Integration

Instead of installing Sub-Apps as npm dependency. We host the subApps file code that would be fetched during run time by the main application.

Build time & size decreases drastically

Remote can be deployed independently at any time

Common dependency can be shared

Complex setup & integration
